What is the purpose of the RWD? Why it is used? How things are done before RWD? What is the difference?

Purpose of RWD is to create flexible webpages that will look good on all devices using CSS Media Queries (the number and popularity of which is quite big nowadays). Evolution and growth of usage of these devices for web-purposes is the main reason of using RWD. That's why nowadays the way of "mobile-first" development became essential. Before RWD things were more desktop-oriented, the most responsiveness were applied with fluid grids to make the content scalable. But this approach doesnt work as good for lower resolutions and smaller viewports. The main difference between ways of RWD and Fluid layout is that the first one uses CSS Media Queries to show different layouts based on screen sizes and types, whereas the second one is based on laying out the content so, that elements take up the same percentage of space depending on screen size.

Images are not directly inserted into a page, they are linked to the page. The ' < 'img'>' tag creates a holding space for the referenced image, whereas the content of the ' < 'figure'>' element is related to the main flow, but its position is independent of that and if figure element is removed, it doesntt affect the flow of the document.
